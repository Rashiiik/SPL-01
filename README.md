# Image Processing CLI in C

A command-line BMP image processing application written in C. The project supports scaling, denoising, sharpening, edge detection, and basic utility filters with optional multithreading.

## What It Does

The program loads a BMP image, opens an interactive menu, applies selected operations, and writes results to an output BMP file.

Main capabilities:

- Scaling
  - Bilinear interpolation
  - Lanczos interpolation
- Denoising
  - Median filter
  - Non-local means
- Sharpening
  - Laplacian filter
  - Unsharp mask
- Edge detection
  - Sobel operator pipeline (grayscale + blur + sobel)
- Utilities
  - Grayscale conversion
  - Gaussian blur
  - Transpose/rotate
  - Negative/invert
- Runtime options
  - Adjustable thread count (1-20)
  - Status view with image metadata and ASCII preview
- Performance logging
  - Operation timings appended to `Log.csv`

## Project Structure

- `src/`: implementation files
- `include/`: header files
- `obj/`: generated object files
- `Makefile`: build and clean targets
- `Log.csv`: benchmark log output

## Requirements

- C compiler with C99/C11 support (`gcc` recommended)
- `make`
- Windows or Linux terminal
- Input image in `.bmp` format

Notes:

- The code includes Windows-specific console API (`windows.h`), so Windows is the primary target.
- The provided `Makefile` clean command uses Windows shell commands (`rmdir`, `del`).

## Installation / Build

From the project root:

### Windows (MinGW/MSYS2)

```bash
make
```

This creates the executable `pixel.exe`.

### Linux/macOS

You can still compile with `gcc`, but you may need to remove or guard Windows-only code first (for example `windows.h` and console color calls in `src/main.c`).

## Run

Usage:

```bash
./pixel <input.bmp> <output.bmp>
```

On Windows CMD/PowerShell:

```powershell
.\pixel.exe .\input.bmp .\output.bmp
```

Example:

```powershell
.\pixel.exe .\lena.bmp .\output.bmp
```

After launch, use the interactive menu to apply operations.

## Menu Overview

Top-level menu:

1. Scaling
2. Denoise
3. Sharpening
4. Edge Detection
5. Utilities
6. Status
7. Multithreading
8. Quit

Each operation can prompt for parameters (for example kernel size, sigma, amount, output dimensions).

## Output and Logs

- Processed image is saved to the output path passed on startup.
- Timing and execution metadata are appended to `Log.csv` in this format:
  - `Operation,Width,Height,Threads Used,Time Taken (s)`

## Clean Build Artifacts

On Windows:

```bash
make clean
```

This removes object files and the executable generated by `make`.

## Troubleshooting

- If `make` is not recognized: install MinGW/MSYS2 and ensure `make` and `gcc` are in your `PATH`.
- If build fails on non-Windows systems: patch/remove Windows-specific console code in `src/main.c`.
- If output file is not updated: verify the input BMP path and permissions, then rerun with a valid output path.
